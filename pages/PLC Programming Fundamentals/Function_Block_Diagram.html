Function Block Diagram (FBD)

Function Block Diagram (FBD) is a graphical PLC programming language defined under the IEC 61131-3 standard. In FBD, the control logic is represented using interconnected function blocks that perform specific operations. Each block represents a function, and the overall control logic is created by connecting the outputs of some blocks to the inputs of others. This visual representation makes FBD highly suitable for process control and applications involving analog signals.

FBD is widely used in industrial automation systems where continuous control, signal processing, and modular programming are required. It allows engineers to design control logic in a structured and intuitive manner.

Basic Elements of FBD

  1. Function Blocks:
    • Perform operations like logic, arithmetic, timers, counters, etc.
    • Examples: AND, OR, NOT, ADD, SUB, TON (timer on delay).
  2. Inputs:
    • Variables or signals that enter a block.
    • Can be digital (TRUE/FALSE) or analog (numbers).
  3. Outputs:
    • Result of a block’s operation.
    • Can be connected to other blocks or physical outputs (like motors, lamps).
  4. Connections (Lines):
    • Show the signal flow from inputs → function blocks → outputs.

Working of Function Block Diagram

During PLC operation, the FBD program is executed in the sequence defined by the block connections. The PLC evaluates each function block using the input values stored in memory and computes the output values accordingly.

Unlike Ladder Diagram, FBD does not strictly follow rung-based execution. Instead, execution depends on data dependencies between blocks. The PLC ensures that blocks are executed in the correct order to maintain logical consistency.

If you are unfamiliar with “AND” Logic, it is actually pretty simple. Basically, you need at least two inputs energized to turn on or energize its output. I like to think of it as you need both inputs (input 1 and input 2) to turn on the output.

Applications of Function Block Diagram

Function Block Diagram is commonly used in:

  • Process control systems.
  • Analog signal processing.
  • PID control loops.
  • Temperature, pressure, and flow control.
  • Continuous industrial automation processes.
