NO and NC

Normally Open (NO) and Normally Closed (NC) are fundamental logical elements used in PLC programming to represent input conditions and internal logic states. These terms describe the logical behavior of contacts in a PLC program and are derived from traditional electrical control systems.

A Normally Open (NO) contact represents a condition that is FALSE when the associated input or variable is inactive. When the input becomes active, the NO contact changes its state to TRUE, allowing logic continuity. In PLC programs, NO contacts are commonly used to represent start buttons, sensors, or conditions that should activate an output when they become true.

A Normally Closed (NC) contact represents a condition that is TRUE when the associated input or variable is inactive. When the input becomes active, the NC contact opens logically and becomes FALSE. NC contacts are widely used in safety circuits such as emergency stop buttons and fault detection logic, where a broken wire or inactive signal should result in stopping the process.

When you create a new boolean instruction in your PLC program you will give it a certain address. This is the address of a single bit (boolean is 0 or 1 and so is a single bit).

Once you’ve given the instruction an address, the state of that particular bit will now be represented as the state of that instruction.

For example you can give an instruction the address of one of the PLC inputs. When doing so, the state of the instruction will now represent the state of the input. Because each PLC input has its own bit in the PLC memory.

The state of each of these bits represents the state of the corresponding input.

Example of NO and NC Contact

This ladder diagram illustrates a basic Start-Stop motor control circuit implemented in a PLC.

  • Inputs:
    • Start push button connected to input I0.0.
    • Stop push button connected to input I0.1.
  • Memory Bit:
    • M10.0 is an internal relay or memory bit used to maintain the motor’s running state (also called a seal-in or holding contact).
  • Output:
    • Q0.0 controls the motor or lamp indicator.

Operation:

  1. When the Start button (I0.0) is pressed:
    • The normally open contact labeled "Start" (I0.0) closes.
    • This energizes memory bit M10.0 which latches itself through its own contact M10.0 in parallel with the Start button.
    • This holding or seal-in circuit keeps M10.0 energized even after the Start button is released.
  2. The output coil Q0.0 (motor/lamp) is energized as long as:
    • The Start condition is latched (M10.0 closed), AND
    • The Stop button (I0.1) is NOT pressed (represented by the normally closed contact I0.1).
  3. Pressing the Stop button (I0.1):
    • Opens the Stop contact, breaking the circuit.
    • De-energizes the memory bit M10.0.
    • De-energizes the output Q0.0, stopping the motor or turning off the lamp.
