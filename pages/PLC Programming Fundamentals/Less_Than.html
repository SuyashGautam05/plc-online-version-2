Less Than

A Less Than ( < ) Compare Instruction in PLC programming is used to compare two values and determine whether one value is smaller than the other.

  • Function: Evaluates a condition by comparing two operands to check if the first value is less than the second.
  • Output: Boolean (TRUE or FALSE) depending on whether the comparison condition is satisfied.
  • Type:
    • Less Than ( < ): Checks whether IN1 < IN2.

Applications

  • Monitoring process parameters to ensure they remain within minimum limits.
  • Activating control actions when a sensor value drops below a setpoint.
  • Checking whether a counter or timer value is below a predefined threshold.

Examples

  • Compare a tank level with the minimum required level.
  • Check if room temperature is lower than the desired set temperature.
  • Determine whether a conveyor speed is below the required operating speed.

Syntax / Representation

In LAD/FBD:

  • The instruction is represented as a comparison block:

|--[ < ]--|

  • Inputs:
    • IN1: First value (e.g., sensor input)
    • IN2: Second value (e.g., reference or setpoint)
  • Output:
    • Q: TRUE if IN1 < IN2, otherwise FALSE

Move Instructions

The MOV (Move) instruction is used to copy data from one memory location to another. It does not modify the source value but transfers its contents to a destination register.

  • The MOV instruction is commonly used for:
    • Data transfer between registers.
    • Updating setpoints.
    • Copying sensor values.
    • Preparing data for display or communication.
  • MOV instructions simplify data handling and improve program readability by clearly defining data flow within the PLC program.

Syntax / Representation

In LAD/FBD:

  • The instruction is represented as a move block:

|--[ MOV ]--|

  • Operands:
    • Source (IN): Value or memory location from which data is copied
    • Destination (OUT): Memory location where the value is stored

Applications

  • Transferring sensor data to internal memory registers.
  • Copying setpoint values for processing or comparison.
  • Updating output registers based on calculated results.
  • Moving values between different data types or memory areas.

Examples

  • Move the value of an analog input into a data register.
  • Copy a preset value into a timer or counter.
  • Transfer a calculated result to an output word.
