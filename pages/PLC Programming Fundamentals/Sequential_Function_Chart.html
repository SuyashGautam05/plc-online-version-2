Sequential Function Chart

A Sequential Function Chart (SFC) is a graphical programming language used in programmable logic controllers (PLCs) to model sequential processes. It is part of the IEC 61131-3 standard for PLC programming.

  • Purpose: To represent control processes that occur in steps and transitions.
  • Key idea: Divide a process into steps (states) and transitions (conditions to move between steps).

In simpler terms, it’s a flowchart-like diagram for controlling sequential operations in industrial automation.

What is a sequence in SFC?

A sequence is a step-by-step operation that should be used to perform a task. An example of a sequence can be a part painting process.

Here are the steps in a part painting process.

  1. The first step is placing the part that needs to be painted in the machine.
  2. And next step is the first coat of paint.
  3. The next step is drying.
  4. And the last step is the final coat of paint.

Let’s see how we can implement these steps easily using SFC programming. An SFC makes it easy to program the sequence directly in the PLC.

An SFC program is composed of graphic symbols. Each graphic symbol represents one step. A step is the same as a state.

Components of SFC

SFCs are built using a few basic elements:

  1. Steps (States):
    • Represent a particular operation or state in the process.
    • Can be active or inactive.
    • Symbol: Rectangle.
  2. Transitions:
    • Logical conditions that determine whether the next step should activate.
    • Symbol: Horizontal line with a condition written on it.
  3. Actions:
    • Operations performed when a step is active.
    • Can be of types:
      • Continuous action: Active as long as step is active.
      • Set/Reset action: Changes output only once when step becomes active.
      • Pulse action: Action occurs briefly when step becomes active.
  4. Connections / Flow Lines:
    • Show the direction of process flow from one step to another.
  5. Initial Step:
    • The first step that is active when the process starts.
    • Marked with a double rectangle or arrow.

Working of SFC

The working of SFC involves activating steps and checking transitions:

  1. Start with the initial step.
  2. Execute the action(s) of the active step.
  3. Check the transition conditions connected to that step.
  4. If the transition condition is true, deactivate the current step and activate the next step.
  5. Repeat this process until the final step is reached.

Key points:

  • Multiple steps can be active in parallel (parallel branches).
  • Transitions only allow step change when conditions are satisfied.

SFC example

Let’s look at an SFC example. We want a lamp to flash at 2 secondsinterval.

To program the SFC we start with the init step.

We need to add two steps. The sequence includes one step for the lamp to turn on, and another step for the lamp to turn off.

  • There is always a transition between each step.
  • If a condition always is full fit just enter TRUE as shown between Init and Step1.
  • When right after entering Init step the program will go to Step1.
  • The program needs to be repeated, because of the continuous flashing.
  • Draw a line from Step 2 to the end before step 1.
  • The lamp needs to flash at 2 seconds intervals.
  • Therefore, a time delay of 2 seconds between the steps is needed.
  • The on-time and the off-time are the same therefore same delay time is used.
  • Then the action to control the lamp is to be added.
  • The qualifier letter N is used because the lamp must be turned on only in Step1.
  • The lamp turns automatically off when leaving Step1 when using the qualifier letter N.
  • No action is needed for Step2 because the lamp is turned off when entering step 2.
  • The variable name or TAG name is Lamp and is assigned to the output module where the physical lamp is wired.
  • Another solution for implementing a flashing lamp is to use the qualifier S to turn the lamp on and the qualifier R to turn the lamp off.
